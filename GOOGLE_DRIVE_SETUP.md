# Googleドライブ連携のセットアップ手順

「☁️ Drive保存 / Drive復元」を動かすには、GoogleのOAuthクライアントID（無料）が必要です。
一度設定すれば、利用者は自分のGoogleアカウントで許可するだけで使えます。

**データの流れ**: 利用者のブラウザ → 利用者本人のGoogleドライブ。
サーバーやデータベースは使わず、運営者が利用者のデータに触れることはありません。
権限も「このアプリが作成したファイルのみ」（`drive.file`スコープ）に限定されます。

---

## 手順1: Google Cloud プロジェクトを作る

1. https://console.cloud.google.com/ にGoogleアカウントでログイン
2. 画面上部のプロジェクト選択 →「新しいプロジェクト」→ 名前は `shukatsu-note` など → 作成

## 手順2: APIを2つ有効にする

1. 左メニュー「APIとサービス」→「ライブラリ」
2. 「Google Sheets API」を検索 →「有効にする」
3. 同じく「Google Drive API」を検索 →「有効にする」

## 手順3: OAuth同意画面を設定する

1. 左メニュー「APIとサービス」→「OAuth同意画面」（新UIでは「Google Auth Platform」→「ブランディング」）
2. User Type は「**外部**」を選択
3. アプリ名（例: 就活ノート）、サポートメール、開発者連絡先を入力して保存
4. **テストユーザー**に自分のGmailアドレス（例: go.mm.8324@gmail.com）を追加
   - テスト段階では、ここに登録した人だけが使えます

## 手順4: OAuthクライアントIDを作る

1. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」
2. アプリケーションの種類: 「**ウェブアプリケーション**」
3. **承認済みのJavaScript生成元**に次の2つを追加:
   - `http://localhost:5173` （ローカルテスト用）
   - `https://yasunishimaki.github.io` （公開サイト用。パスは付けない）
4. 作成すると「クライアントID」（`〜.apps.googleusercontent.com` で終わる文字列）が表示されるのでコピー

## 手順5: クライアントIDをアプリに設定する

`src/driveBackup.js` の冒頭にある次の行に貼り付けます:

```js
const GOOGLE_CLIENT_ID = "ここにクライアントIDを貼る";
```

※ クライアントIDは公開しても問題ない識別子です（パスワードや秘密鍵ではありません）。
Publicリポジトリにコミットして大丈夫です。

## テスト手順（ローカル）

```bash
npm install
npm run dev   # http://localhost:5173 を開く
```

1. **保存テスト**: ヘッダーの「☁️ Drive保存」→ Googleログイン・許可 → 「保存しました」表示
   → 自分のGoogleドライブに「就活ノート バックアップ」というスプレッドシートができていることを確認
   （企業・ES・予定・メモの各シートに内容が表として入っています）
2. **上書きテスト**: アプリでデータを少し変更 → もう一度「Drive保存」
   → 新しいファイルは増えず、同じスプレッドシートが更新されることを確認
3. **復元テスト**: ブラウザの開発者ツール（F12）→ コンソールで `localStorage.clear()` を実行 → ページを再読み込み
   → 初期サンプルデータに戻る → 「☁️ Drive復元」→ 保存した内容が戻ることを確認

## よくあるエラー

| 症状 | 原因と対処 |
|---|---|
| ポップアップが出ない | ブラウザのポップアップブロック。アドレスバーの通知から許可する |
| `redirect_uri_mismatch` / `origin_mismatch` | 手順4の「JavaScript生成元」のURLが間違っている。ポート番号・httpsの有無を確認 |
| `access_denied`（このアプリはテスト中…） | 手順3のテストユーザーに自分のアドレスが入っていない |
| 403エラー | 手順2のAPI有効化漏れ（Sheets/Driveの両方が必要） |

## 公開時の注意（他の人にも使ってもらう場合）

- 同意画面が「テスト」のままだと、**テストユーザー以外は使えず、許可も7日で切れます**
- 「OAuth同意画面」→「アプリを公開」で**本番環境**に切り替えると誰でも使えるようになります
- 使用スコープが `drive.file`（非機密）のみなので、**Googleの審査は不要**で、警告画面も出ません
